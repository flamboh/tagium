Cobalt load-test baseline

This is a historical sizing baseline, not a current capacity guarantee. The test used real downloads against a disposable, single-vCPU Fly Machine with Cobalt's native rate limit raised so Machine saturation could be observed.

  • Roughly 20–45 concurrent downloads completed comfortably.
  • Latency degraded noticeably around 100 concurrent downloads.
  • At roughly 150–200, requests formed multi-minute backlogs.

A real Tagium download includes one resolve request and one or two tunnel requests for audio and cover art. Those results informed the separate resolve and tunnel gates in cobalt-machine-proxy.mjs.

Re-run bun run load-test:cobalt against a disposable clone whenever the Machine shape, Cobalt version, provider mix, or proxy implementation changes. Never load-test the production deployment.
