Implementation plan · Cobalt resource protection

Cancellation-safe download admission

Prevent rapid start/cancel cycles from resetting Tagium's local download budget, propagate cancellation to Cobalt, and enforce a server-side ceiling before work reaches Fly.

Prepared 2026-07-09 · Applies to the queue and resilience work in PR #66 · Companion to the Cobalt proxy architecture

Decision Evidence Architecture Semantics Implementation Testing Rollout Acceptance

Decision

Treat a Cobalt plan start as an admitted resource operation. Admission history belongs to the browser session and the server, not to a disposable queue run. Cancellation stops work where possible, but never refunds a permit after the plan request crosses the network seam.

Browser

One session ledger

Keep the 60-second reservation window alive across cancel, retry, and replacement runs.

Transport

One abort chain

Carry browser cancellation through the Worker to the Fly proxy and upstream Cobalt.

Server

One authoritative gate

Reject excessive plan starts before Fly allocates work, independent of client behavior.

Reproduced failure

A deterministic controller probe started three tracks, canceled immediately, and repeated the sequence seven times without advancing the clock. The intended client ceiling is 40 tunnel units per minute, or 20 ordinary tracks at the default cost of two.

Why the budget resets

createPlaylistDownloadQueueRun creates a new queue model and empty reservation list. The controller deliberately creates a fresh run when the previous run is canceled.

Why cancel does not undo server work

requestCobaltAudio uses its own timeout signal instead of combining it with the incoming request signal. The Worker can therefore continue its upstream request after the browser has stopped waiting.

Current failure path

1 Queue starts Three tracks reserve budget inside run A and begin plan requests.
2 User cancels Fibers are interrupted and run A becomes canceled.
3 Queue restarts Run B is created with an empty reservation ledger.
4 Worker continues The upstream fetch has only a timeout signal, not the client abort.
5 Fly absorbs work Capacity gates bound concurrency, but not starts per actor per minute.

Target architecture

Introduce two deep modules at deliberate seams. Queue callers keep their existing small interface; timing, pruning, cancellation, and server adapter details stay behind the modules.

In-process module

DownloadAdmissionWindow

Owned by the long-lived playlist controller. It hides the sliding-window ledger, cost accounting, pruning, wake scheduling, and abortable waiting. 

admit({ cost, now, signal })
  → admitted
  → waits until capacity exists
  → aborts without spending if still waiting
Remote-owned seam

CobaltRequestAdmission

Called by the audio route before its Fly fetch. A Cloudflare rate-limiting adapter runs in production; an in-memory adapter makes the same interface deterministic in tests and local dev. 

admit({ sessionKey, clientKey })
  → allowed
  → rejected { status: 429, retryAfter: 60 }
  → unavailable { status: 503 }
1 Queue requests admission All runs share one browser-session window.
2 Permit is spent Only when the plan request is ready to dispatch.
3 Worker admits actor Session limit first, coarse client/IP backstop second.
4 Fly gates capacity Existing resolve and tunnel concurrency gates remain unchanged.
5 Abort travels upstream Cancel closes Worker fetch, Fly request, and Cobalt work where possible.

Cancellation and permit semantics

State at cancellation Permit treatment Runtime action User-visible result
Pending behind local admission Do not consume Remove the waiter and cancel its wake timer. Track becomes canceled immediately.
Admitted, before fetch invocation Conservatively retain Abort before network I/O if the signal wins the dispatch race. No retry loop; reservation expires normally.
Plan request dispatched Retain until window expiry Abort browser fetch and propagate the signal to the Worker upstream fetch. Track is canceled; no permit refund.
Tunnel or local processing active Retain until window expiry Close the Worker/Fly chain; existing proxy cleanup destroys upstream. Partial work is discarded safely.
Server admission rejected Client permit remains spent Return 429 with Retry-After; pause rather than immediate retry. Queue explains the wait and resumes later.

Limits are intentionally layered

Local admission

Prevents accidental abuse and gives immediate queue feedback. Reloading can reset it.

Worker admission

Protects Fly from a reset or bypassed client. It is the resource-policy authority.

Fly capacity gates

Protect the single shared vCPU from concurrency spikes; they do not replace rate limits.

Cloudflare's Rate Limiting binding is appropriate for abuse mitigation, but its counters are permissive, eventually consistent, and local to a Cloudflare location. If telemetry shows that a strict global ceiling is necessary, replace only the production adapter with a Durable Object or a trusted keyed limiter at the single Fly proxy; callers keep the same interface.

Implementation sequence

Make the client ledger survive runs

  1. Extract budget arithmetic from queue item state.
  2. Own one admission window in the controller closure.
  3. Carry reservations across cancel and retry.
  4. Keep the controller interface unchanged.

Propagate cancellation end to end

  1. Pass the incoming request signal into Cobalt resolution.
  2. Combine it with the five-minute timeout.
  3. Classify client abort separately from upstream failure.
  4. Verify Fly destroys the upstream request.

Add authoritative Worker admission

  1. Add session and coarse client rate-limit bindings.
  2. Admit before calling Fly.
  3. Return 429 and Retry-After on denial.
  4. Remove the production in-memory Map limiter.

Observe and tune

  1. Log coarse admission outcomes without URLs.
  2. Watch 429, abort, and Fly resolve counts.
  3. Compare admitted starts with upstream starts.
  4. Adjust thresholds only from production evidence.

Initial policy

Gate Initial threshold Key Reason
Browser admission 40 tunnel units / 60 seconds20 ordinary tracks at cost 2 Controller lifetime Preserves the current product limit while closing cancel/restart reset.
Worker session admission 20 plan starts / 60 seconds Server-issued anonymous session cookie Matches ordinary local throughput and survives queue replacement or page logic errors.
Worker client backstop 60 plan starts / 60 seconds Coarse Cloudflare client key Limits session-token churn while leaving room for shared networks.
Fly resolve gate Existing concurrency and queue caps Global Machine capacity Protects CPU and latency independently of actor admission.

Test plan

Client regression tests

  • Seven start/cancel cycles at one timestamp start exactly 20 tracks, not 21.
  • The 21st track enters a waiting state and starts when the oldest reservation expires.
  • Canceling a waiter consumes no permit and leaves no wake fiber.
  • Canceling after dispatch retains the permit.
  • Retry and a newly loaded SoundCloud set share the same admission history.
  • Advancing the clock prunes only expired reservations.

Server and transport tests

  • The 21st session plan request returns 429 without invoking the Fly fetch.
  • The coarse client backstop rejects rotating session identifiers at its threshold.
  • Rate-adapter failure returns 503 and does not fail open to Fly.
  • Aborting the incoming request aborts the upstream fetch signal.
  • Timeout still aborts when the browser remains connected.
  • Fly proxy cleanup releases its gate once and destroys upstream on disconnect.

Rollout and PR structure

PR A · stack on #66

Client admission

Land the reproduced regression test and session-owned browser ledger first. This closes the easy accidental bypass without changing infrastructure.

PR B · stack on PR A

Abort propagation

Wire the incoming signal through the Worker and prove that cancellation reaches the Fly proxy. This reduces wasted work independently of rate policy.

PR C · stack on PR B

Server admission

Add bindings, production policy, 429 handling, and observability. Exercise it in preview before promoting to production.

Preview verification

  1. Set a low temporary session threshold in preview.
  2. Run start/cancel cycles and verify no Fly call after denial.
  3. Confirm queue copy shows a wait instead of a generic failure.
  4. Inspect logs for counts only; URLs and track metadata must remain absent.

Rollback

  1. Server adapter can be disabled by configuration if false positives appear.
  2. Keep browser admission active; it has no infrastructure dependency.
  3. Abort propagation is independently safe to retain.
  4. Do not restore the old per-run ledger as a fallback.

Observability and privacy

Emit only coarse outcomes: admitted, waited, canceled-before-dispatch, canceled-after-dispatch, server-rate-limited, and upstream-started. Include duration, threshold class, and deploy SHA; exclude source URLs, filenames, titles, and raw client keys.

Primary ratio

upstream starts / admitted starts should approach 1 and never exceed it materially.

Waste signal

Track how many upstream starts are aborted before a plan is returned.

False-positive signal

Track unique sessions receiving 429 and whether they succeed after Retry-After.

Acceptance criteria

  • Canceling and immediately restarting cannot reset the browser's 60-second ledger.
  • The 21st ordinary track waits rather than starting another server request.
  • Waiting requests can be canceled without spending admission.
  • Dispatched requests remain charged even when canceled.
  • Browser cancellation reaches the Worker upstream fetch.
  • The Fly proxy destroys upstream work and releases capacity once.
  • Server admission runs before any Fly request.
  • Server denial returns 429 with Retry-After and does not auto-retry immediately.
  • Production does not depend on an isolate-local JavaScript Map for enforcement.
  • Logs and analytics contain no media URLs, titles, filenames, or raw actor keys.
  • Existing queue cancellation, retry, hydration, and stale-run tests continue to pass.
  • Preview validation demonstrates the limiter without touching production Cobalt.

Explicit non-goals and rejected shortcuts

Button debounce Useful polish, but it reduces click frequency rather than enforcing resource starts.
Wait for old fibers Prevents overlap but still allows unlimited sequential starts within the minute.
Refund on abort Unsafe after dispatch because Cobalt may already have spent CPU or contacted an upstream provider.
IP-only identity Shared networks create false positives; use a session key with a higher coarse client backstop.
Capacity gate only Concurrency protects latency and memory, not per-actor consumption over time.
